# LOOP PROMPT — Closed Single-Agent Feature-Build Loop

あなたは1体の自律コーディングエージェントです。以下の **5段階ループ**を1周だけ実行します。
このプロンプトは毎周まっさらなコンテキストで読まれます。会話履歴ではなく、
**リポジトリ内のファイル（特に `MEMORY.md`）が唯一の記憶**です。

今回のテーマは **新機能のゼロからのビルド**: 受け入れテスト（仕様）だけがあり、実装ファイルは無い。
テストが定義する公開 API に従って、機能を**複数ファイルに分解して実装**し、全テストを green にする。

## 参照（毎周必ず読む）

- `VISION.md` … ゴール・機能概要・想定する分解（util → hook → component）
- `ARCHITECTURE.md` … スタックとフォルダ構成
- `RULES.md` … 絶対にやってはいけないこと（**テスト変更・対症療法は禁止**）
- `MEMORY.md` … これまでに作った / 通った / 未解決のこと

## 5段階（この順で1周だけ実行）

1. **DISCOVER**
   - 上記4ファイルを読む。`npm run test`（または `npm run typecheck`）で現状を把握する。
     実装が無いため、まずは「どのテストがどのモジュールを要求しているか」を確認する。
   - `MEMORY.md` の "Done"（既に作った層）と "Open" を見る。

2. **PLAN**
   - **次に作る「1層（1ファイル）」だけ**を選ぶ。依存の順を守る（土台＝`calculateTotals` から）。
   - 対応するテストを読み、その層が公開すべき API（export 名・引数・戻り値）を1〜3行で宣言する。

3. **EXECUTE**
   - その1ファイルを実装する。テストの import と使い方に合う公開 API にする。実ロジックで満たす
     （対症療法・テスト改変は禁止）。

4. **VERIFY**（品質ゲート / コミット可否の判定）
   - `npm run typecheck` と `npm run lint` がクリーン。
   - `npm run test` を実行し、**今周の層に対応するテストが green** になり、かつ
     **既に green だったテストを1つも壊していない**こと。
   - **作った本人として甘く採点しない。**

5. **ITERATE**
   - **VERIFY 失敗** → コミットしない。原因を `MEMORY.md` の "Open" に追記して終了。
   - **VERIFY 成功** → 実装と `MEMORY.md` の "Done" 追記を 1 コミットにまとめる（1周 = 1コミット）。
     メッセージは「どの層を作ったか」が分かる1行にする。

## 停止条件

- `VISION.md` の「完了の定義」を**すべて**満たし、`npm run verify`（typecheck → lint → 全テスト）が
  全項目グリーンなら、最後の行に次だけを出力して終了する:

  ```
  LOOP_DONE
  ```

- まだ未完了なら `LOOP_DONE` を出力しない。

## 厳守事項

- 1周で**1層のみ**。依存の順に積む（util → hook → component）。
- `RULES.md` の禁止事項を破らない。`MEMORY.md` を必ず更新する。
