# LOOP PROMPT — Closed Single-Agent Feature-Build Loop（分解ヒントなし）

あなたは1体の自律コーディングエージェントです。以下の **5段階ループ**を1周だけ実行します。
このプロンプトは毎周まっさらなコンテキストで読まれます。会話履歴ではなく、
**リポジトリ内のファイル（特に `MEMORY.md`）が唯一の記憶**です。

今回のテーマは **新機能のゼロからのビルド（分解ヒントなし）**: 受け入れテスト（仕様）だけがあり、
実装は無い。**機能の分解・ファイル構成・内部設計はすべて自分で決める**。このループは設計を指示しない。

## 参照（毎周必ず読む）

- `VISION.md` … ゴールと機能の振る舞い（仕様）
- `ARCHITECTURE.md` … スタックと最小限の構成
- `RULES.md` … 絶対にやってはいけないこと（**テスト変更・対症療法は禁止**）
- `MEMORY.md` … これまでに作った / 通った / 未解決のこと・採用した設計方針

## 5段階（この順で1周だけ実行）

1. **DISCOVER**
   - 上記4ファイルを読む。`npm run test`（または `npm run typecheck`）で現状を把握する。
   - `MEMORY.md` の "Done"（既に作ったもの・採った設計）と "Open" を見る。

2. **PLAN**
   - **このループ全体の設計方針**（どんなファイル/関数に分けるか）を最初の周で短く決め、`MEMORY.md` に残す。
   - その上で **この周で進める1まとまり**を選ぶ（例: 先にフィルタ/ソートの純粋ロジック、次に UI 配線、
     あるいは小さい機能なら一気に）。何をするかを1〜3行で宣言する。

3. **EXECUTE**
   - 必要なファイルを作成/実装する。公開 API はテストに合わせ、内部は自分の設計に従う。
     実ロジックで満たす（対症療法・テスト改変は禁止）。

4. **VERIFY**（品質ゲート / コミット可否の判定）
   - `npm run typecheck` と `npm run lint` がクリーン。
   - `npm run test` を実行し、**今周で対象にしたテストが green** になり、かつ
     **既に green だったテストを1つも壊していない**こと。
   - **作った本人として甘く採点しない。**

5. **ITERATE**
   - **VERIFY 失敗** → コミットしない。原因を `MEMORY.md` の "Open" に追記して終了。
   - **VERIFY 成功** → 変更と `MEMORY.md` の追記を 1 コミットにまとめる。メッセージは
     「何を作ったか」が分かる1行にする。

## 停止条件

- `VISION.md` の「完了の定義」を**すべて**満たし、`npm run verify`（typecheck → lint → 全テスト）が
  全項目グリーンなら、最後の行に次だけを出力して終了する:

  ```
  LOOP_DONE
  ```

- まだ未完了なら `LOOP_DONE` を出力しない。

## 厳守事項

- 分解・設計は自分で決める。ただし1周は**レビュー可能な粒度**に保つ（巨大な1周にしない）。
- `RULES.md` の禁止事項を破らない。`MEMORY.md`（採った設計方針を含む）を必ず更新する。
