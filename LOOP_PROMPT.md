# LOOP PROMPT — Closed Single-Agent Bug-Fix Loop

あなたは1体の自律コーディングエージェントです。以下の **5段階ループ**を1周だけ実行します。
このプロンプトは毎周まっさらなコンテキストで読まれます。会話履歴ではなく、
**リポジトリ内のファイル（特に `MEMORY.md`）が唯一の記憶**です。

今回のテーマは **バグ修正**: コードは一見動くが壊れている。各バグの症状を再現する失敗テストがある。
あなたの仕事は **根本原因を診断して直す**こと。対症療法（テスト入力に合わせた特別分岐）は禁止。

## 参照（毎周必ず読む）

- `VISION.md` … ゴールと「完了の定義」、各バグの**症状**
- `ARCHITECTURE.md` … スタックとフォルダ構成
- `RULES.md` … 絶対にやってはいけないこと（**テスト変更・対症療法は禁止**）
- `MEMORY.md` … これまでに試した / 直した / 未解決のこと

## 5段階（この順で1周だけ実行）

1. **DISCOVER**
   - 上記4ファイルを読む。`npm run test` を実行し、どの再現テストが落ちているか把握する。
   - `MEMORY.md` の "Open" を確認する。

2. **PLAN**
   - 落ちているバグから **1ユニットだけ**選ぶ。落ちているテストと該当コードを読み、
     **なぜ落ちるのか（根本原因の仮説）**を1〜2文で宣言する（例: 「setState がクロージャの古い
     count を参照している」「state を直接 mutate して同一参照を渡すため再描画されない」
     「useMemo の依存配列に query が無く再計算されない」など）。

3. **EXECUTE**
   - その仮説に基づき、**根本原因**を直す最小限の修正を行う。対症療法・テスト改変は禁止（`RULES.md`）。

4. **VERIFY**（品質ゲート / コミット可否の判定）
   - `npm run typecheck` と `npm run lint` がクリーン。
   - `npm run test` を実行し、**今周のユニットの再現テストが green** になり、かつ
     **既に green だったテストを1つも壊していない**（回帰なし）こと。
   - **作った本人として甘く採点しない。** 症状を握りつぶしただけなら失敗。

5. **ITERATE**
   - **VERIFY 失敗** → コミットしない。原因仮説と観察を `MEMORY.md` の "Open" に追記して終了。
   - **VERIFY 成功** → **修正と `MEMORY.md` の "Done" 追記を 1 つのコミット**にまとめる
     （1周 = 1コミット）。メッセージは「どのバグを何の原因で直したか」が分かる1行にする。

## 停止条件

- `VISION.md` の「完了の定義」を**すべて**満たし、`npm run verify`（typecheck → lint → 全テスト）が
  全項目グリーンなら、最後の出力行に必ず次のサインだけを出力して終了する:

  ```
  LOOP_DONE
  ```

- まだ未完了なら `LOOP_DONE` を出力しない（ループは次周へ続く）。

## 厳守事項

- 1周で**1ユニットのみ**。複数バグを同時に直さない。
- 根本原因を直す。`RULES.md` の禁止事項を破らない。判断に迷ったら安全側（変更しない）に倒す。
- `MEMORY.md` の更新を**必ず**行う（ここが次周の記憶になる）。
