# LOOP PROMPT — Closed Single-Agent Refactor Loop

あなたは1体の自律コーディングエージェントです。以下の **5段階ループ**を1周だけ実行します。
このプロンプトは毎周まっさらなコンテキストで読まれます。会話履歴ではなく、
**リポジトリ内のファイル（特に `MEMORY.md`）が唯一の記憶**です。

今回のテーマは **リファクタリング**: コードは既に動作しテストは全緑。あなたの仕事は
**振る舞いを変えずに**、ESLint の構造ルールを満たすようコードを綺麗にすること。
テストは駆動条件ではなく「壊していないか」の**回帰ネット**として常に全緑を保つ。

## 参照（毎周必ず読む）

- `VISION.md` … ゴールと「完了の定義」（lint 全通過 + テスト全緑のまま）
- `ARCHITECTURE.md` … スタックとフォルダ構成
- `RULES.md` … 絶対にやってはいけないこと（最優先で遵守。**eslint-disable 禁止・テスト/設定変更禁止**）
- `MEMORY.md` … これまでに試した / 通った / 未解決のこと

## 5段階（この順で1周だけ実行）

1. **DISCOVER**
   - 上記4ファイルを読む。`npm run lint` を実行して残っている違反（ファイル・ルール・箇所）を把握する。
   - `npm run test` が全緑であること（リファクタ前の基準）を確認する。`MEMORY.md` の "Open" を見る。

2. **PLAN**
   - lint 違反が残るユニットから **次に着手する「1ユニット」だけ**を選ぶ。
   - どのルール違反を、どんな構造変更（関数抽出・ガード節・記述子マップ等）で解消するかを1〜3行で宣言する。
     振る舞いは変えないこと。

3. **EXECUTE**
   - そのユニットだけをリファクタする。**公開シグネチャと振る舞い（出力・レンダリング結果）は保つ**。
   - `eslint-disable` やルール緩和で誤魔化さない（`RULES.md`）。

4. **VERIFY**（品質ゲート / コミット可否の判定）
   - `npm run typecheck` がクリーン。
   - `npm run lint` が **今周のユニットについて違反 0**（他ユニットの既存違反は残っていてよい）。
   - `npm run test` が **全 21 テスト緑のまま**（1つも壊していない・回帰なし）。
   - **作った本人として甘く採点しない。** 振る舞いを変えてテストを通す/壊すのは失敗。

5. **ITERATE**
   - **VERIFY 失敗**（typecheck/対象ユニットの lint が赤い・テストを1つでも壊した）
     → コミットしない。原因仮説を `MEMORY.md` の "Open" に追記して終了（次周で再挑戦）。
   - **VERIFY 成功** → **リファクタの変更と `MEMORY.md` の "Done" 追記を 1 つのコミット**にまとめる
     （1周 = 1コミット）。コミットメッセージは「どのユニットを何のルール充足のために整理したか」を1行で。

## 停止条件

- `VISION.md` の「完了の定義」を**すべて**満たし、`npm run verify`（typecheck → lint → 全テスト）が
  全項目グリーンなら、最後の出力行に必ず次のサインだけを出力して終了する:

  ```
  LOOP_DONE
  ```

- まだ未完了なら `LOOP_DONE` を出力しない（ループは次周へ続く）。

## 厳守事項

- 1周で**1ユニットのみ**。複数ユニットを詰め込まない。
- 振る舞い不変。`RULES.md` の禁止事項を破らない。判断に迷ったら安全側（変更しない）に倒す。
- `MEMORY.md` の更新を**必ず**行う（ここが次周の記憶になる）。
