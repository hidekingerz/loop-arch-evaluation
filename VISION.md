# VISION — このループのゴール

> 検証テーマ: **バグ修正（再現テスト起点の診断 → 修正）**
> エージェントは毎周これを読み、「完了の定義」を満たしたかどうかで停止を判断します。

## ゴール（1〜2文で）

`src/` 配下の **一見動作しているが実は壊れているフロントエンド実装** に対し、症状を再現する
**失敗テスト**が用意されている。各バグの**根本原因を診断して修正**し、再現テストを含む全テストを
green にする。テストファイルは変更しない。

## バグ報告（症状）

- **`src/components/StepCounter.tsx`** … 「N つ進める」ボタンを1回押しても **1 しか増えない**
  （本来は step ぶん増えるはず）。
- **`src/components/TaskToggle.tsx`** … チェックボックスを操作しても **UI と「完了: N」カウンタが
  更新されない**（見た目が変わらない）。
- **`src/lib/useFilteredList.ts`** … 初回の絞り込みは正しいが、**クエリを変えても結果が変わらない**
  （古い結果のまま）。

> 症状だけを示している。**原因は自分で診断**すること（対症療法でなく根本原因を直す）。

## 完了の定義（Definition of Done） — 検証可能な箇条書きで

- [ ] `npm run typecheck`（`tsc --noEmit`）がエラー 0
- [ ] `npm run lint`（`eslint .`）がエラー・警告 0
- [ ] `npm run test`（`vitest run`）が **全 13 テスト pass**（再現テスト＋既存テスト）
- [ ] `npm run verify` が成功する
- [ ] テストファイル（`src/**/*.test.ts(x)`）を一切変更していない

## 対象ユニット（1周＝1ユニットが目安）

- `src/components/StepCounter.tsx`
- `src/components/TaskToggle.tsx`
- `src/lib/useFilteredList.ts`

## スコープ外（やらないこと）

- テストファイルの変更・削除・スキップ。
- **対症療法**（テストの入力値に合わせた特別分岐・症状の握りつぶし）。根本原因を直すこと。
- 既存で緑のテストが守る振る舞いを壊すこと。公開 API シグネチャの変更。
- 依存パッケージの追加・更新、ビルド設定の変更、`templates/single-agent-loop/` の変更。
