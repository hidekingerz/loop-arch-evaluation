# RULES — エージェントが絶対にやってはいけないこと

> 対象リポジトリ向けに書き換えてください。VERIFY の品質ゲートと並ぶ「ガードレール」です。
> エージェントは PLAN / EXECUTE でこのルールを最優先で守り、抵触する場合は変更しません。

## 禁止事項（Hard rules）

- {{例: 本番設定ファイル（.env.production, config/prod.*）を変更しない}}
- {{例: 依存パッケージの追加・削除・バージョン変更をしない}}
- {{例: VERIFY を通っていない変更をコミットしない}}
- {{例: 既存テストを「通すために」削除・スキップしない}}
- {{例: 大規模リファクタ（1周で多数ファイルにまたがる変更）をしない}}
- {{例: 秘密情報・認証情報をコードやログに書かない}}

## 迷ったときの原則

- 判断に迷ったら**安全側（変更しない）**に倒し、`MEMORY.md` の "Open" に判断ポイントを記録する。
- 1周 = 1タスク。スコープを広げない。
