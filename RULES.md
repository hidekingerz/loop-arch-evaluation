# RULES — エージェントが絶対にやってはいけないこと

> VERIFY の品質ゲートと並ぶ「ガードレール」です。PLAN / EXECUTE でこのルールを最優先で守ります。

## 禁止事項（Hard rules）

- テストファイル（`src/**/*.test.ts(x)`）を**変更・削除・スキップしない**。テストは仕様。
- **対症療法をしない**。テスト入力に合わせたハードコード（特定の合計値・商品 id を直接返す等）で
  通すのは禁止。任意の入力に一般化する実ロジックで実装する。
- 依存パッケージの追加・削除・変更、ビルド/ツール設定
  （`tsconfig.json` / `eslint.config.js` / `vitest.config.ts` / `package.json`）の変更をしない。
- VERIFY を通っていない変更をコミットしない。
- 1周で複数ユニットにまたがる実装をしない（1周 = 1ファイル/層が目安）。
- `templates/single-agent-loop/` を変更しない。

## 迷ったときの原則

- 公開 API はテストの import と使い方に合わせる。テストに無い内部構造は素直で読みやすい設計にする。
- 依存の順（util → hook → component）に積む。土台が無いと上位層のテストは緑にできない。
- 1周 = 1層。スコープを広げない。判断に迷ったら `MEMORY.md` の "Open" に残す。
