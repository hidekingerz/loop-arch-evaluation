# RULES — エージェントが絶対にやってはいけないこと

> 品質ゲート（lint）と回帰ネット（test）を両立させるためのガードレール。
> PLAN / EXECUTE でこのルールを最優先で守ります。

## 禁止事項（Hard rules）

- **振る舞いを変えない**。`validateRegistration` の入出力（エラーメッセージ文言も）、
  `StatusBadgeList` のレンダリング結果（`data-testid`・テキスト・要素構造）を変更しない。
- テストファイル（`src/**/*.test.ts(x)`）を**変更・削除・スキップしない**。テストは仕様であり回帰ネット。
- **`eslint-disable` / `eslint-disable-next-line` / `// eslint-ignore` 等でルールを黙らせない**。
  実際にコード構造を改善して満たす。
- `eslint.config.js` のルールを**緩和・削除・変更しない**。`tsconfig.json` / `package.json` /
  `vitest.config.ts` も変更しない。
- 依存パッケージの追加・削除・バージョン変更をしない。
- 公開エクスポートのシグネチャ（型・引数・戻り値の形）を変えない。新しいヘルパは追加してよいが、
  既存の公開関数/コンポーネントの外形は保つ。
- VERIFY を通っていない変更をコミットしない。1周で複数ユニットにまたがる変更をしない。
- `templates/single-agent-loop/` を変更しない。

## 迷ったときの原則

- 「綺麗にする」ためでも**振る舞いを変えそうなら止める**。安全側（テストが緑のまま通る変更）に倒す。
- 判断に迷ったら `MEMORY.md` の "Open" に記録する。1周 = 1ユニット。
