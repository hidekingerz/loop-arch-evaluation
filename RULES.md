# RULES — エージェントが絶対にやってはいけないこと

> maker・checker の両方が最優先で守るガードレール。

## 禁止事項（Hard rules）

- テストファイル（`src/**/*.test.ts(x)`）を**変更・削除・スキップしない**。
- **対症療法をしない**。`if (totalSeconds === 65) return "1:05"` のような**テスト入力に合わせた
  ハードコード分岐**で症状だけ消すのは禁止。**任意の入力に一般化する根本修正**を行う。
- 既存で緑のテストが守る振る舞いを壊さない。公開シグネチャ（引数・戻り値の形）を変えない。
- 依存パッケージの追加・削除・変更、ビルド/ツール設定の変更をしない。
- checker が承認していない（または VERIFY を通っていない）変更をコミットしない。
- `templates/single-agent-loop/` を変更しない。

## 迷ったときの原則

- 「この修正は `formatDuration(125)` など**テストに無い入力**でも正しいか？」を必ず自問する。
  No なら対症療法。一般化する修正に直す。
