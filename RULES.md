# RULES — エージェントが絶対にやってはいけないこと

> VERIFY の品質ゲートと並ぶ「ガードレール」です。PLAN / EXECUTE でこのルールを最優先で守ります。

## 禁止事項（Hard rules）

- テストファイル（`src/**/*.test.ts`, `src/**/*.test.tsx`）を**変更・削除・スキップしない**
  （`it.skip` / `describe.skip` / `it.only` / コメントアウトも含む）。テストは仕様であり不可侵。
- テストを「通すために」**期待値に合わせたハードコード分岐**を実装に入れない（実際のロジックで満たす）。
- 依存パッケージの**追加・削除・バージョン変更**をしない。
- ビルド/ツール設定（`tsconfig.json`, `eslint.config.js`, `vitest.config.ts`, `package.json`）を変更しない。
- `templates/single-agent-loop/`（雛形のオリジナル）を変更しない。
- VERIFY（`npm run verify`）を通っていない変更をコミットしない。
- 1周で複数ユニットにまたがる大規模変更をしない（1周 = 1ユニットが目安）。
- 秘密情報・認証情報をコードやログに書かない。

## 迷ったときの原則

- 判断に迷ったら**安全側（変更しない）**に倒し、`MEMORY.md` の "Open" に判断ポイントを記録する。
- テストが曖昧/矛盾して見えても、テストではなく実装側で満たす方法を探す。
