# RULES — エージェントが絶対にやってはいけないこと

> VERIFY の品質ゲートと並ぶ「ガードレール」です。PLAN / EXECUTE でこのルールを最優先で守ります。

## 禁止事項（Hard rules）

- テストファイル（`src/**/*.test.ts(x)`）を**変更・削除・スキップしない**。テストは仕様。
- **対症療法をしない**。テスト入力（特定の商品名・価格・並び順）に合わせたハードコードや
  テスト専用の分岐で通すのは禁止。任意の入力に一般化する実ロジックで実装する。
- 依存パッケージの追加・削除・変更、ビルド/ツール設定
  （`tsconfig.json` / `eslint.config.js` / `vitest.config.ts` / `package.json`）の変更をしない。
- VERIFY を通っていない変更をコミットしない。
- `templates/single-agent-loop/` を変更しない。

## 迷ったときの原則

- 公開 API（`ProductList` の props・テストが触る要素）はテストに合わせる。**内部設計は自由**。
- フィルタ/ソートのロジックは一般化して書く（テストに無い入力でも正しく動くこと）。
- 判断に迷ったら `MEMORY.md` の "Open" に残す。
